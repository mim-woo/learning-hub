# 데이터 분석 방법론

> ADsP 2과목 데이터 분석 기획에 기반한 학습 내용

> 단순한 내용 정리보단 실제 프로젝트에서 가지는 의미를 기반으로 해석 및 정리

## 0. 데이터 분석 기획의 이해

**1). 분석주제 유형**

- 분석 대상과 분석 방법에 따라 4가지로 분류

- 이는 필요에 따라 융합되거나 반복될 수 있음

*(1). 최적화 (Optimization) : 대상과 방법을 모두 이해한 상태에서 수행 / 현 상태를 최적화하는 방법*

*(2). 솔루션 (Solution) : 분석 방법을 알지 못할 때 수행 / 분석 과제를 수행하며 솔루션을 탐색*

*(3). 통찰 (Insight) : 대상이 불분명하나 방법은 알고 있을 때 수행 / 주어진 데이터를 기반으로 인사이트를 도출*

*(4). 발견 (Discover) : 대상과 방법을 모두 모를 때 수행 / 분석 대상 자체를 도출하는 과정*

**2). 분석 기획 방안**

- 목표 시점에 따라 '과제중심적 접근방식'과 '장기적 마스터플랜 방식'을 나누어 기획 방안을 결정

- 이 둘을 적절히 융합하여 종합적인 기획 방안을 도출 해야 함

  *과제 중심적 접근방식 : Speed & Test / Quick-Win / Problem Solving*
  
  *장기적 마스터플랜 방식 : Accuracy & Deploy / Long Term / Problem Defintion*

- 단계마다 방식을 적용할수도 있고, 과제 단위 해결을 통해 1차적 분석을 진행할 수도 있음

- 분석 기획시 고려 사항

  (1). 가용 데이터
    분석을 위한 데이터 확보는 필수적. 유형에 대한 분석이 선행되어야 솔루션과 분석 방법을 결정 가능
  
    *정형 데이터 (Structured Data) : 고정된 컬럼과 같이 행과 열에 의해 속성이 구별되는 데이터 ex) 스프레드시트 등 / 스키마 구조를 가져 탐색 과정이 정형화되어있음*
    *반정형 데이터 (Semi-Structured Data) : 데이터 내부에 스키마에 해당하는 메타데이터가 존재, 일반적으로 파일 형태 / 데이터 형태를 파악하는 과정이 필요 / ex) URL, HTML, XML 등*
    *비정형 데이터 (Unstructured Data) : 데이터 세트가 아니라 각각의 데이터가 객체화된 상태 / 웹에 존재하는 경우 html 형태를 띌 수 있으나 텍스트마이닝되는 경우도 있으므로 명확한 구분 어려움 / ex) 멀티미디어 데이터*
  
  (2). 적절한 유스케이스 분석 : 유사한 분석 시나리오 및 솔루션 활용 / 사용자 측면에서의 공감대 확보

  (3). 장애요소에 대한 사전계획 : 정확도와 분석 비용의 상관관계 등 분석과정에서 발생할 수 있는 장애요소에 대한 사전 대응 계획을 수립 및 고려해 분석을 진행
